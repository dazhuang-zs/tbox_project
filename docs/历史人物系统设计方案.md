# 历史人物系统设计方案

## 一、项目概述

### 1.1 项目背景
基于用户反馈，当前系统存在以下问题：
- 历史人物数据量不足
- 缺乏人物背景故事和历史事件关联
- 缺少人物能力数据和成就数据
- 没有系统化的评估体系

### 1.2 设计目标
本方案旨在构建一个完整的历史人物系统，实现：
- 丰富的历史人物数据库（覆盖中国历史主要人物）
- 沉浸式的故事叙述体验
- 量化的能力评估体系
- 多维度的人物对比功能

### 3.2 能力评估体系

#### 3.2.1 能力维度定义
共设计8个核心能力维度：

| 维度 | 说明 | 评分标准 |
|------|------|----------|
| 政治能力 (Political) | 治国理政、政治谋略 | 0-100分，含描述 |
| 军事能力 (Military) | 用兵、战略、战术 | 0-100分，含描述 |
| 文学造诣 (Literary) | 诗词文章、文学创作 | 0-100分，含描述 |
| 艺术才能 (Artistic) | 绘画、音乐、书法等 | 0-100分，含描述 |
| 战略眼光 (Strategic) | 长远规划、大局观 | 0-100分，含描述 |
| 个人魅力 (Charisma) | 人格魅力、感染力 | 0-100分，含描述 |
| 创新能力 (Innovation) | 思想创新、改革魄力 | 0-100分，含描述 |
| 领导力 (Leadership) | 团队管理、组织能力 | 0-100分，含描述 |

#### 3.2.2 评分算法

**综合评分计算公式：**
```
total_score = Σ(ability[i].score × weight[i]) / Σ(weight[i])

权重设置：
- 政治家：political=0.25, leadership=0.20, strategic=0.15, charisma=0.15
- 军事家：military=0.30, strategic=0.25, leadership=0.20
- 文学家：literary=0.40, artistic=0.20, innovation=0.15
- 艺术家：artistic=0.35, literary=0.20, innovation=0.15
```

**等级划分：**
- S级：90-100分（千古一帝/绝世奇才）
- A级：80-89分（一代伟人/杰出人物）
- B级：70-79分（著名人物）
- C级：60-69分（知名人物）
- D级：60分以下（普通人物）

不同时代对能力的权重需要进行调整：

## 四、故事系统设计

### 4.1 故事类型

#### 4.1.1 人物故事线索
- **成长故事**：从出生到成名的历程
- **转折故事**：人生关键时刻
- **成就故事**：重大成就的详细过程
- **情感故事**：友情、爱情、亲情
- **传奇故事**：民间传说与野史

#### 4.1.2 故事关联机制
```
人物 → 生平事件 → 历史大事件 → 其他人物
  ↓
关系网络 → 故事线交织
```

### 4.2 故事渲染模板

**叙事风格：**
```markdown
## 【{人物名} · {时代标签}】

> {人物名言或名句}

### 生平概览
{summary}

### 🎯 关键时刻
{key_moments}

### 💫 历史成就
{achievements}

### 🔗 关联人物
- **{关系类型}**：{人物名} - {关系描述}
- ...

### 📜 后世评价
{evaluations}
```
