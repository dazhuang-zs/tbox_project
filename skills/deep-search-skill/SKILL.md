# 深度搜索技能

此技能用于指导 AI 执行完整的深度搜索流程，包括多轮搜索和信息整合，最终生成专业的 Markdown 格式搜索报告。

## 核心流程

### 1. 理解需求与规划

首先，阅读主提示词文件了解完整工作流程：
```
references/prompts/deep-search-main-prompt.md
```

该文件包含：
- 用户需求总结与介绍
- 项目信息分析
- 搜索策略规划（深度搜索、广度搜索、数据搜索）
- 多轮搜索执行
- 搜索结果报告生成

### 2. 执行搜索

**关键要求：必须使用 MCP 搜索服务进行搜索**

搜索通过 MCP (Model Context Protocol) 服务调用实现。在 openclaw 环境中，你应该：

1. **调用 MCP 搜索工具**：使用'web-search.web-search'MCP搜索服务进行网络搜索
   - 搜索关键词应明确、具体
   - 每轮可以并发进行多个搜索（最多3个）
   - 支持深度搜索、广度搜索、数据搜索等不同类型

2. **搜索轮次要求**：
   - 简单主题：1轮搜索即可
   - 调研类问题：至少3轮，最多6轮搜索
   - 每轮搜索后需要向用户汇报收获和下一步方向

3. **搜索类型**：
   - **深度搜索**：深入某个相关子主题
   - **广度搜索**：扩展当前主题，覆盖更多相关领域
   - **数据搜索**：搜索可用于图表展示的数据

### 3. 生成搜索报告

搜索完成后，需要生成专业的搜索报告文档：

**步骤**：

1. **阅读写入提示词**：
   ```
   references/prompts/deep-search-write-file-prompt.md
   ```
   
2. **整理搜索结果**：
   - 汇总所有搜索轮次的信息
   - 提取关键信息和数据
   - 标注信息来源（URL）

3. **生成 Markdown 文档**：
   - 文件名格式：`search_{主题名称}.md`
   - 文件名使用中文，避免特殊字符
   - 使用脚注格式标注信息来源
   - 结构化呈现信息

4. **写入文件到工作目录**：
   - 将生成的 Markdown 文档写入当前工作目录

## 执行示例

### 用户请求示例
```
帮我搜索 Kotlin 协程的相关信息，包括基础概念、最佳实践和实际应用案例
```

### 执行流程

1. **需求分析与介绍**
   ```
   我将为您进行 Kotlin 协程的深度搜索，包括：
   - 基础概念和核心原理
   - 最佳实践指南
   - 实际应用案例
   预计进行3-4轮搜索，最后生成详细的搜索报告。
   ```

2. **第一轮搜索**（广度搜索）
   - 使用 MCP 搜索："Kotlin 协程 基础概念"
   - 使用 MCP 搜索："Kotlin 协程 教程"
   - 使用 MCP 搜索："Kotlin 协程 官方文档"
   
   **搜索完成消息**：
   ```
   **第一轮搜索完成**
   
   收获：成功获取 Kotlin 协程官方文档及中文翻译版本，包括核心概念、基础教程和 Android 应用场景。
   
   **下一步：深入搜索结构化并发、CoroutineScope 和 Context 相关内容。**
   ```

3. **第二轮搜索**（深度搜索）
   - 使用 MCP 搜索："Kotlin 协程 CoroutineScope"
   - 使用 MCP 搜索："Kotlin 结构化并发"
    
4. **第三轮搜索**（数据搜索）
   - 使用 MCP 搜索："Kotlin 协程 性能对比数据"
   - 使用 MCP 搜索："Kotlin 协程 使用统计数据"

5. **生成搜索报告**
   - 读取 `references/prompts/deep-search-write-file-prompt.md`
   - 整合所有搜索结果
   - 生成 `search_Kotlin协程详解.md`
   - 写入文件到工作电脑

## 注意事项

### 搜索阶段
- ✅ 必须使用 MCP 搜索服务
- ✅ 每轮搜索后向用户汇报进展
- ✅ 保持专业和简洁的沟通风格
- ✅ 合理规划搜索轮次和并发数量
- ❌ 不要输出完整的搜索报告文本（在搜索阶段）
- ❌ 不要使用 emoji 和复杂 markdown 格式

### 报告生成阶段
- ✅ 必须标注所有信息来源（URL）
- ✅ 使用脚注格式引用来源
- ✅ 文件名使用 `search_` 前缀
- ✅ 文件名使用中文，避免特殊字符
- ✅ 内容结构化、专业化
- ❌ 不要添加欢迎词或多余说明
- ❌ 不要遗漏原始链接

## 关键要点

1. **必须使用给定的提示词文件**：两个提示词文件已经定义了完整的搜索和写作流程，不要修改或忽略
2. **MCP 搜索服务**：通过 MCP 调用搜索功能，而不是其他方式
3. **文件写入**：最终产物是 Markdown 文件，写入到电脑工作目录
4. **用户体验**：在搜索过程中保持与用户的沟通，告知进展和方向
5. **专业性**：生成的报告要结构化、信息全面、来源清晰
